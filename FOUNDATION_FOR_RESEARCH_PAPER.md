## 1. OBJECTIVE

Turn an ordinary **photograph of furniture** into a **BIM-compliant IFC model**, and scale that to
**populating whole buildings** with correctly-sized, ergonomically-placed furniture that can be
visualized in the browser (xeokit) and opened in professional BIM tools (Revit/ArchiCAD).

Two products, one asset pipeline:
1. **Per-item:** photo → detected object → real-world-measured → 3D mesh → IFC.
2. **Building-scale:** load a real architectural IFC (rooms/walls/doors) → auto-populate every room
   by type → one populated building (IFC/XKT/GLB) → xeokit.

Commercial framing: **SCS sells outputs (IFC files + configured rooms), not model code/weights.** So
licences that bind only model *redistribution* (MIT/Apache/SAM) don't restrict what SCS ships;
licences that bind *outputs* or impose revenue caps (CC-BY-NC, Stability, Tencent) are avoided.

---

## 2. IDEAS PUSHED (evolution of the concept)

1. **Single-image → 3D** via generative models (TripoSR/TRELLIS/etc.).
2. **Pivot to retrieval + metric grounding:** detect (DETR) → measure real size (Depth-Anything-v2
   metric) → retrieve a real catalog mesh (DINOv2 over Amazon Berkeley Objects). Generative used as
   fallback. This makes outputs *dimensionally real* and *commercially clean*.
3. **BIM export:** every object written as IFC4 with real geometry + hierarchy.
4. **Ergonomic room layout:** a CP-SAT solver places furniture per standards (Neufert/Panero/ADA).
5. **Asset library + instancing:** each mesh stored once; buildings are placement *tables*.
6. **Building-scale population:** aggregate rooms → whole building.
7. **Load real architecture:** the key realization — *our tool populates furniture, not architecture*;
   so we load a real building IFC (Duplex) as the container and auto-populate its rooms.
8. **In-progress:** upload-your-own generated IFC with auto-categorization + a "generated by us"
   visual badge.

## 3. LIMITATIONS (honest)

- **6 GB GPU ceiling.** Live mesh generation (DETR+Depth+TripoSR) is at the edge of an RTX-4050;
  repeated runs poison system state and it **segfaults (0xC0000005 / exit 139) on both GPU and CPU**
  until a **reboot**. All CPU tools (solver, IFC, XKT, populate) are unaffected.
- **The tool does not model architecture.** No walls-with-thickness, doors, handles, or non-box
  rooms. Architecture must come from a real IFC (proof: our building IFC has 0 IfcSpace/walls/doors).
- **Rectangular room assumption.** The solver + populate use each room's bounding box; L-shaped rooms
  are approximated.
- **Orientation outliers.** Generators emit inconsistent up-axes; real-world scaling is best-effort
  (e.g. a bed once mis-sized). Fix = per-item PCA up-axis detection.
- **XKT is a JSON fallback,** not binary (the `gltf2xkt` CLI isn't installed) — larger/slower than
  optimal; still loads.
- **Retrieval quality varies by category** (see §10 scores — some F-scores are low).

## 10. KEY CONCEPTS + FORMULAS

**F-score (3D reconstruction accuracy, primary metric).** For predicted point set P and ground-truth
G at distance threshold τ:
- Precision = |{p ∈ P : min_g‖p−g‖ < τ}| / |P|
- Recall    = |{g ∈ G : min_p‖g−p‖ < τ}| / |G|
- **F = 2·Precision·Recall / (Precision + Recall)**

**Chamfer Distance (companion metric).**
CD(P,G) = (1/|P|) Σ_p min_g‖p−g‖² + (1/|G|) Σ_g min_p‖g−p‖²

**Benchmark result (H200, 5 generators + TripoSR variants; per-category winners in the asset library):**
overall mean F ranking ≈ **TripoSG 0.393 > SAM 3D 0.368 > TRELLIS 0.347 > InstantMesh 0.327 > TripoSR**.
No single model wins every category → the library uses **best-of-each** (stool 0.99 TripoSG,
table 0.81 InstantMesh, cabinet 0.73 SAM 3D, bed 0.67 TRELLIS, …).

**Metric dimensioning.** Depth-Anything-v2-metric gives per-pixel metric depth d(u,v). Back-projecting
the detected mask to a metric point cloud via intrinsics K, the object's real size is the extent of
that cloud: (H,W,D) = max−min over axes. (Enables true-scale IFC instead of unit blobs.)

**Ergonomic layout (CP-SAT).** Decision vars = each item's (x,z,rotation) on the floor. Constraints:
no-overlap of AABBs with per-category clearance buffers c; wall margin; door keep-clear ≥ leaf depth;
min aisle (Neufert ≥1.0 m); ADA (route 0.915 m, door 0.815 m, turning circle 1.525 m). Objective:
place all items / maximize an ergonomic score. Item-agnostic revision: clearance and wall-affinity are
now **derived from geometry** (tall h≥1.2 m or aspect≥1.6 → perimeter; low+square → free).

**Geometry instancing (why buildings render fast).** N placements of one asset cost **1 geometry +
N transforms** (O(1) geometry memory) vs N embedded copies (O(N)). Example: 47 instances from 9 meshes.

**Quadric decimation (BIM practicality).** Iterative edge collapse minimizing the quadric error
Q = Σ (nᵀv + d)² (squared distance to incident planes). Raw 150k–2.7M faces → ~8–10k; 10-item IFC
138 MB → 2.4 MB.

**Coordinate transform (IFC ↔ glTF).** IFC is **Z-up** (floor = XY plane); glTF/xeokit is **Y-up**.
Conversion = rotation −90° about X: **(x,y,z) → (x, z, −y)**. Assets and IFC are both Z-up, so
furniture drops into rooms with no per-item rotation; one final scene rotation makes it upright in
the viewer.

## 11. THE IFC WORK (detailed)

- **Write:** `saveIFC.py` / `createIFCFurniture.py` emit **IFC4** — `IfcProject→Site→Building→Storey`
  hierarchy, `IfcTriangulatedFaceSet` geometry (`IfcCartesianPointList3D` verts + faces),
  `IfcFurniture`/`IfcChair` per object, real placement (position, scale; rotation baked into mesh).
  Re-opened + validated with IfcOpenShell every run.
- **An `.ifc` is TEXT (STEP / ISO-10303-21).** The geometry lives inside as coordinate lists — it is
  *not* an image. It renders only in a viewer. (This was a core point of confusion; proven by opening
  a generated chair IFC in Autodesk Viewer.)
- **IFC → 3D space (3 paths; all loaders present in the vendored xeokit SDK):**
  1. IFC → **XKT** (`convert_to_xkt.py`) → `XKTLoaderPlugin` (fast, BIM-proper; recommended).
  2. IFC → **GLB** (`ifc_to_glb.py`, IfcOpenShell tessellation) → `GLTFLoaderPlugin` (simplest; used
     for the building viewers).
  3. IFC → `WebIFCLoaderPlugin` (raw `.ifc` in-browser).
- **Building IFC:** `build_building_ifc.py` writes one IFC4 from a placement table (world transforms,
  rotation baked).
- **Read (new, for auto-populate):** `populate_building.py` reads every `IfcSpace` (room name +
  footprint) from a real building IFC — segmentation is *already in the IFC*, no analysis needed.

## 14. AREAS FOR OPTIMIZATION

**Code / performance**
- Install `@xeokit/xeokit-convert` → **binary XKT** (10–100× faster load than the JSON fallback).
- **Viewer instancing:** load asset geometry once via xeokit `SceneModel` + per-instance transforms
  (currently building GLBs embed copies).
- **Non-rectangular rooms:** clip furniture to the true `IfcSpace` footprint polygon (not bbox).
- **PCA up-axis detection** to auto-correct orientation outliers.
- **Serialize generation + free VRAM** between runs (torch.cuda.empty_cache; a job queue) to avoid the
  6 GB stacking crash; offer a CPU-safe retrieval-only mode.
- Cache DINOv2/DETR/depth models across requests instead of reloading per subprocess.

**Security**
- **Upload validation** (the in-progress dropbox): restrict to `.ifc`/`.glb`, size-cap, sanitize
  filenames, validate magic bytes, scan for path traversal before any `send_from_directory`.
- No auth on `:8000`/`:3000` — fine for localhost demo; add auth + HTTPS before hosting.
- Validate/limit all API inputs (item counts, room dims) server-side (partly done).
- Avoid shell-injection in Python subprocess spawns (use arg arrays — already the pattern).
- LGPL-3.0 (IfcOpenShell) + CC-BY-4.0 (ABO) notices must ship with any distribution.

## 16. ONE-PARAGRAPH ABSTRACT (seed for the paper)

We present a pipeline that converts a single photograph of furniture into a dimensionally-accurate,
BIM-compliant IFC model, and scales it to automatic whole-building furnishing. Rather than rely solely
on single-image 3D generation, we ground objects in reality: DETR detection + Depth-Anything-v2 metric
depth recover true dimensions, and DINOv2 retrieval selects a real, measured mesh from the Amazon
Berkeley Objects catalog, with generative models (benchmarked across TripoSG, SAM 3D, TRELLIS,
InstantMesh, TripoSR by F-score/Chamfer) as fallback. Objects are exported as validated IFC4 and placed
by an OR-Tools CP-SAT solver that encodes ergonomic standards (Neufert/Panero/ADA). Crucially, we show
that architecture (walls/doors/rooms) should come from a real IFC while our system supplies the
furniture: loading a standard building (Duplex) and reading its `IfcSpace` rooms, we auto-populate every
room by type in seconds on CPU, rendered in real time via xeokit geometry instancing.
