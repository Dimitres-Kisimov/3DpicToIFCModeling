IFC size reduction — ninety percent lighter, still valid IFC4

This before/after compares the same bed embedded in IFC. The raw photo-derived mesh carries 150,000 triangles in a 5.1 MB file; after the IFC optimizer it carries 15,000 triangles in 420 KB — a 90% reduction in faces and 92% in file size — while remaining a valid IFC4 object with visually equivalent geometry. It demonstrates that reconstructed meshes must be decimated before IFC embedding for models to remain openable and shareable in standard BIM tools.

Why it exists & what it documents: An early IFC would not open in standard viewers because the embedded mesh was too heavy, so we had to quantify how much the IFC optimizer recovers. It documents that a ~90%/92% reduction yields a valid, openable IFC4 — the justification for running the optimizer automatically on every export.
