🌳 Program Structure

how the code is organised — click any branch to open it

← Research hub
The idea in one sentence: a Node server is the front door, Python workers do all heavy computation as isolated subprocesses, and every placement decision comes from one ergonomics rulebook — so each part can be tested, replaced or extended on its own. The full engineering map with extension recipes lives in the Developer Manual.
backend/ SERVER — the application layer
server.js — Express entry point: serves the app, the research hub, and mounts every API route
routes/ — one file per API area
apiRoutes.js — /api/generate: photo upload → engine → repaired mesh → auto-catalog
roomRoutes.js — catalog, items, custom-category upload, room layout, deletes
buildingRoutes.js — building registry, IFC upload/probing, rooms/floors, populate, save
services/ — the plumbing
pythonBridge.js — spawns Python scripts, passes JSON in/out, enforces timeouts
roomApi.js — cached gateway to room_api.py commands
GPU / CPU queues — heavy jobs are serialized so bursts can't overload the machine
python-scripts/ COMPUTE — where the intelligence lives
run_triposr.py — the photo→3D pipeline: segmentation (rembg), TripoSR, post-processing
graft_chair_base.py — chair base repair: 6 parametric base styles + broken-bottom auto-detection
rule_packs.py — THE ergonomics rulebook: Neufert / Panero / ADA / ASR (German workplace law), archetypes, clearance zones
spatial_layout.py — the CP-SAT constraint solver: places furniture with no overlaps, clear zones, circulation
populate_building.py — whole-building population: room classification (6 languages), density tiers, walking-path protection, rotated-building handling, the presentation-hall row engine
room_api.py — the command hub the server calls: rooms, registration, numbering, deletes
catalog.py — the 38-category furniture catalog (ABO + engine-generated + your custom uploads)
saveIFC.py — IFC4 export + the validation gate every object must pass
make_tower_ifc.py — synthesize tall buildings from any floor plate
test_e2e_smoke.py · test_floor_dissection.py — the two permanent test gates
frontend/ BROWSER — what you're looking at
index.html + js/index.js — the three workspaces: Generate · Build a room · Building
js/roomBuilder.js — catalog picker, custom categories, capacity guard
js/buildingMode.js — room cards per floor, density tiers, clash engine, floor isolation
js/buildingPlan.js — the 2D floor plan editor (drag with live legality checks)
hub.html · research_roadmap.html · building_explorer.html — the research hub, the dated roadmap, the 15-building 3D explorer
xeokit viewer — WebGL 3D, loaded locally (no CDN)
data/ DATA — everything the app knows
generated_assets/ — the numbered catalog items (desk-TSG-042 style) + manifest (single source of truth)
buildings/ — the 14-building fleet + registry + per-building geometry cache
mesh_library_abo/ — 515 real furniture meshes + the DINOv2/FAISS retrieval index
benchmark/ EVIDENCE — the science behind the choices
187 photos × 5 engines, A/B lists 01–11, the 998-variant visualizer, per-AI IFC folders
docs/ + deliverable/ — manuals, compliance, provenance, engine guides
docs/DEVELOPER_MANUAL.md — where every functionality lives + how to extend
docs/ASR_COMPLIANCE.md · docs/BUILDINGS_PROVENANCE.md · docs/USER_GUIDE.md
deliverable/manuals/ — 13 AI-engine installation manuals with lessons learned
docs/archive/ — the project's historical reports, kept but out of the way
the flow of one click
📷 photo → apiRoutes → pythonBridge → run_triposr → repair → saveIFC gate → catalog (numbered) → pick it in a room → spatial_layout places it by the rulebook → export IFC4 → open in Revit
