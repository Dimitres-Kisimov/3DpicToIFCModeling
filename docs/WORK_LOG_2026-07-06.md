## PART 1 — Office-chair base graft (DONE)

**Problem:** TripoSR reconstructs a chair's seat/back well but fragments the 5-star swivel base into
~500 floating bits. No cleanup can rebuild geometry that was never generated.

**Solution:** keep the generated seat/back, cut the broken base, graft a clean parametric 5-star base
(hub + gas column + 5 spokes + 5 casters), voxel-fused into one solid so it survives the debris
filter and the IFC optimizer. Office-chairs only, via a UI toggle. Details + renders:
[docs/office_chair_base_graft/README.md](office_chair_base_graft/README.md).

**Bugs found & fixed (each verified by rendering / IFC inspection):**
| Bug | Cause | Fix |
|---|---|---|
| Base exploded ~170× | `VoxelGrid.marching_cubes` returns voxel-index coords | refit solid to true AABB |
| IFC optimizer dropped the base | pymeshfix collapsed multi-part mesh to largest piece | repair **per-component**, recombine (`clean_and_optimize.clean_mesh`) — helps all multi-part objects |
| Chair shown sideways | seat-leveling + X-up canonicalize reoriented the mesh; stacked with the viewer's fixed rotation | **removed** both — keep native frame |
| Legs too wide | base radius 0.55×seat | default → **0.42×seat** (fits within seat) |

**Final result:** exported IFC = **one `IfcFurniture` "Office Chair"** carrying the full merged
geometry (seat/back + 5-star base), base below seat, colour preserved. Verified across multiple
independent app-generated chairs. Env switches: `SCS_GRAFT_CHAIR_BASE`, `SCS_BASE_RADIUS_FRAC`
(0.42), `SCS_CHAIR_SMOOTH` (16).

**IFC optimizer** (separate, already integrated): runs automatically on "Download all as one IFC"
(`exportRoutes.js`) — clean unique meshes once → geometry instancing → precision rounding → zip.
Documented: [docs/IFC_OPTIMIZER.md](IFC_OPTIMIZER.md).

## PART 2 — Ergonomics engine + one app (IN PROGRESS)

Full plan: [docs/PART2_ERGONOMICS_AND_ONE_APP_PLAN.md](PART2_ERGONOMICS_AND_ONE_APP_PLAN.md).

**Key discovery:** the room-builder already exists as a **separate Flask app on :8000**
(`backend/app_server.py` + `demo/app.html`) — it does catalog selection, a space/feasibility check
("not enough space"), both room sources (user-defined + loaded Duplex IFC), and drag-reposition.
Part 2 = **harden the ergonomics engine + merge :8000 into the :3000 app + add a 2D editor.**

### Done this session
1. **A1 — object-agnostic archetype layer** (`rule_packs.py`): 8 placement archetypes
   (worksurface/seating/storage/bed/appliance/on_surface/wall_mounted/free_accent) + `ANTHRO` human
   constants. `archetype_of()` maps known categories and infers unknowns from geometry;
   `placement_profile()` returns human interaction zones (legroom, seat pull-out, door swing, bed
   access) + facing + wall side. Verified on known + unknown objects.
2. **D — GPU job serialization** (`backend/services/gpuQueue.js`): concurrency-1 queue around both
   GPU entry points. **Verified on the RTX 4050 (6 GB):** two simultaneous generations serialized,
   peak VRAM 5.47/6.14 GB (two-at-once would OOM), both succeeded. Makes iterative multi-object
   generation crash-safe on this laptop.
3. **D — catalog recall speed** (`apiRoutes.js`): "Fast — from catalog" was secretly falling back to
   TripoSR generation (similarity 0.13 < 0.18 threshold) → ~53 s. Forced `SCS_RETRIEVAL_THRESHOLD=0`
   for the catalog engine → **53 s → 22 s**, `mesh_source=retrieval`. Remaining 22 s is per-request
   model loading — warm-model worker is the next optimization.

### Hardware / deployment findings (for docs)
- Only **photo→3D generation (TripoSR)** needs a GPU (≥6 GB NVIDIA). Everything else (room builder,
  ergonomics solver, IFC, viewer) is light CPU/WebGL.
- **Hosted mode** (GPU on the server, e.g. Hetzner GEX44): any laptop with a browser works — even a
  MacBook Air. **Local mode**: needs an NVIDIA RTX 6 GB+ (gaming laptop) for generation.
- Dev machine = **RTX 4050 Laptop, 6 GB** (~3.8 GB free at rest). The GPU queue + per-process VRAM
  release are what keep multi-object generation from crashing it.
