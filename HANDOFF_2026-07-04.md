## 1. Objective

Turn a **photo of furniture** into a **BIM-compliant IFC** model, and scale that to **auto-furnishing
whole buildings**: load a real architectural IFC, read its rooms, populate each ergonomically, view in
xeokit, export IFC/GLB. Commercial framing: SCS sells **outputs** (IFC + configured rooms), not model
weights ‚Üí only MIT/Apache/SAM/LGPL/CC-BY components are used (royalties = $0).

## 4. The building-population system (centerpiece ‚Äî how it works)

**Entry:** `localhost:8000` ‚Üí Building dropdown ‚Üí **Duplex Apartment** ‚Üí per-room chips ‚Üí **Populate**.

**Flow:**
```
sample_buildings/Duplex_Architecture.ifc  (real: 21 rooms, walls, doors, 4 storeys, IFC2X3, 2.4 MB)
   ‚îÇ  backend/python-scripts/populate_building.py   (called by app_server.py)
   ‚ñº
 read IfcSpace rooms (name+footprint)  ‚Üí  smart_furnish(type,area)  ‚Üí  extract obstacles
 (internal/party walls, beams, columns, stairs, doors) ‚Üí MERGE overlapping obstacles (shapely) ‚Üí
 CP-SAT solver (spatial_layout.py + rule_packs.py: Neufert/Panero/ADA) places AROUND obstacles ‚Üí
 fit-as-many-as-possible (shrink set until all placed cleanly) ‚Üí 0 clashes
   ‚îÇ  --movable mode ‚Üí shell.glb + one GLB per piece + furniture.json (each piece world pos)
   ‚ñº
 app_server.py /populate returns {shell, pieces[]}; frontend loads them as SEPARATE xeokit models ‚Üí
 click-pick + drag on floor (surface pick ‚Üí worldPos, reload-on-release) ‚Üí /save merges dragged
 positions + shell ‚Üí downloadable building.glb
```

**Verified on Duplex:** ~5‚Äì8 rooms furnished, ~18‚Äì23 pieces, **0 clashes**. Living‚Üísofa/table/lamp/
shelf; Bedroom‚Üíbed/wardrobe/(+study if big); Kitchen‚Üícabinets/table; bathrooms/halls/stairs skipped.

**Backend endpoints (Flask, `backend/app_server.py`):**
- `GET /api/buildings` ¬∑ `GET /api/building/<id>/rooms` (rooms+type+area+suggestion) ¬∑
  `POST /api/building/<id>/populate` (‚Üí shell+pieces) ¬∑ `POST /api/building/<id>/save` (‚Üí merged GLB)
- `POST /api/upload_generated` (drop .glb/.ifc ‚Üí auto-categorize ‚Üí `data/generated_assets/manifest.json`)
  ¬∑ `GET /api/generated/<fn>` ¬∑ plus existing `/api/catalog`, `/api/items/<cat>`, `/api/generate`.

**Frontend:** `demo/app.html` + `demo/app.js` (loaded `?v=14`). Building panel + per-room chips +
drag logic (`bPieces`, `loadBuilding`, pointer pick/drag) + Save + drop-zone/upload + "OURS" badge.

## 7. Technical notes for continuity

- **Coordinates:** IFC is **Z-up** (floor=XY, Z vertical); glTF/xeokit is **Y-up**. Convert with rotation
  ‚àí90¬∞ about X: `(x,y,z)‚Üí(x,z,‚àíy)`. Asset GLBs are Z-up + real-scaled, so furniture drops in with a
  yaw-only rotation; the whole scene is rotated to Y-up at the end for the viewer.
- **Rooms are treated as bounding boxes** (obstacles handle intrusions). Real polygon clipping is a TODO.
- **XKT is a JSON fallback** (the `@xeokit/xeokit-convert` CLI isn't installed) ‚Äî loads, but not binary/fast.
- **Metric F-score** primary eval: `F = 2PR/(P+R)`. Benchmark mean: TripoSG 0.393 > SAM3D 0.368 >
  TRELLIS 0.347 > InstantMesh 0.327 > TripoSR. Asset library = best-of-each.
- **Licenses/royalties = $0** for the shipped stack (MIT/Apache/SAM/LGPL/CC-BY). Traps to keep out:
  Depth-Anything **Large** (CC-BY-NC = non-commercial; only *Small* is Apache), StableFast3D (Stability
  community, $1M cap), Hunyuan3D (Tencent). Compute ~$37‚Äì85/mo pilot ‚Üí <$0.005/room at scale.

## 8. ‚öÝÔ∏è Known issues / gotchas

- **Live Generate is slow (~2 min) and 6 GB-VRAM-bound.** It segfaults (`0xC0000005`/exit 139) if the GPU
  state gets poisoned by heavy stacked runs ‚Äî **reboot fixes it**. Generate ONE at a time; close heavy
  3D tabs if it OOMs. All CPU building/IFC tools are unaffected.
- **Servers die on session cycle** unless launched **detached** (PowerShell Start-Process ‚Äî see ¬ß3).
- **spatial_layout rotated-centre bug** ‚Äî corrected in populate_building, not at source (shared with room app).
- **Not one app yet** ‚Äî see ¬ß9.
